## 介绍
traceUp 资源表示上游追踪分析，对 traceUp 执行 GET 请求，可查找给定弧段或节点的上游弧段和结点。
支持的方法：
- [GET](#GET 请求)：查找给定弧段或节点的上游弧段、节点。
- [HEAD](#HEAD 请求)：检查 traceUp 资源是否存在，或权限是否可以访问 traceUp 资源。
支持的表述格式：RJSON、JSON、HTML、XML、FlatGeobuf。

## HTTP 请求方法
对如下 URI 执行 HTTP 请求，以 rjson 输出格式为例加以说明，其中，supermapiserver 是服务器名。
http://supermapiserver:8090/iserver/services/transportationAnalyst-facilityNet/rest/networkanalyst/WaterNet%40FacilityNet/traceup.rjson
### ](../../root.htm)GET 请求
查找给定弧段或节点的上游弧段、节点。
**请求参数**
对 traceUp 资源执行 GET 请求，需要在 URI 中包含一些参数，如下：
| 名称 | 类型 | 含义 |
| --- | --- | --- |
| edgeID | int | 【与nodeID必填一项】需查找上游设施的弧段 ID 。 |
| nodeID | int | 【与edgeID必填一项】需查找上游设施的结点 ID 。 |
| weightName | String | 权重字段，默认为 SMLENGTH。 |
| isUncertainDirectionValid | boolean | 选择是，表示不确定流向有效，遇到不确定流向时分析继续进行；选择否，表示不确定流向无效，遇到不确定流向将停止在该方向上继续查找。 |
| withIndex | boolean | 当使用 FlatGeobuf 表述时该参数可选。表示返回的表述为 FlatGeobuf 的结果是否包含空间索引，默认为true。 |
| returnFeatures | boolean | 否返回结果要素的详细描述信息。若为 false，只返回结果要素的 ID 集合。 |
**响应结构**
执行上有追踪分析功能，返回的操作结果表述的结构如下：
| 字段 | 类型 | 说明 |
| --- | --- | --- |
| cost | double | 分析结果中的耗费。 |
| edges | int[] | 分析结果弧段 ID 集合。 |
| edgesFeatures | Feature[] | 分析结果弧段详细描述信息。 |
| nodes | int[] | 分析结果中的结点 ID 集合。 |
| nodesFeature | Feature[] | 分析结果中的结点要素的详细描述信息。 |
**响应示例**
对示范数据 WaterNet@FacilityNet 数据集执行上游追踪分析，即对如下 URI 执行 GET 请求：
http://supermapiserver:8090/iserver/services/transportationAnalyst-facilityNet/rest/networkanalyst/WaterNet@FacilityNet/traceup.rjson?nodeID=1&isUncertainDirectionValid=true&returnFeatures=true&weightName=SMLENGTH

### HEAD 请求
返回跟 GET 请求一样的 HTTP 响应头，但是没有响应实体。可以在不必传输整个响应内容的情况下，获取包含在响应消息头中的元数据信息。元数据信息包括媒体类型，字符编码，压缩编码，实体内容长度等。
HEAD 请求可以用来判断 traceUp 资源是否存在，或者客户端是否有权限访问 traceUp 资源。通过对加.<format>的 URI 执行 HEAD 请求，还可以快速判断 traceUp 资源是否支持<format>格式的表述。
