SuperMap iServer 支持根据 ArcGIS Server 服务根地址批量转发 ArcGIS Server 中发布的 REST 地图服务、REST 要素服务、REST 矢量瓦片服务和 REST 地址匹配服务。

您可以通过以下步骤，根据实际需求选择需要转发的服务和对应的接口：

  1. 在服务管理界面首页->快速发布或在服务->服务管理页面，点击快速创建服务，进入发布向导；
  2. 选择数据来源为"批量 ArcGIS REST 服务"，点击下一步；
  3. 输入 ArcGIS REST 服务根地址，支持的输入形式为：https://{ip}:{port}/arcgis/rest/services。此外，您还需要根据服务的公开情况，选择安全认证方式并配置安全认证信息，具体如下：
  • 不使用安全认证：选择后无需进一步配置。
  • 用户名/密码：输入有访问权限的用户名、密码及 ArcGIS Token 服务地址（https://{ip}:{port}/arcgis/tokens）
  • Token：输入已经获取的 Token 字符串。若获取的 Token 是以 HTTP Referer 方式生成的，则还需要填写对应的 HTTP Referer 值

完成以上配置后，点击下一步；

  1. 选择发布的服务类型。iServer 默认提供转发 ArcGIS REST 服务时支持的接口，包括：REST-地图服务、REST-数据服务、REST-地址匹配服务、REST-矢量瓦片服务、WMS 1.1.1服务、WMS 1.3.0服务、WMTS 1.0.0服务、WMTS-CHINA服务、WMTS-ARCGIS服务、WFS 1.0.0服务、WFS 2.0.0服务、OGC API 数据服务、OGC API 地图服务、ArcGIS REST 地址匹配服务、Baidu REST 地图服务和 Google REST 地图服务。您可以根据实际需求选择其他需要发布的服务类型和对应的接口。点击下一步；
  1. 进行服务配置：
  • 对于地图服务：选择是否启用地图瓦片缓存。勾选启用时，可指定地图浏览过程中生成的瓦片数据的存储类型（如 UGCV5、SMTiles、MongoDB、OTS（过时））和存储位置，以及缓存是否只读、是否关闭瓦片重采样、缓存过期时间等配置；
  • 对于数据服务：设置数据服务是否允许编辑

完成以上配置后，点击下一步；

  1. 发布完成后，可以看到当前发布的服务名和链接，点击可直接访问服务。
