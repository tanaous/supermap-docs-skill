  <p class="myNormal">作为 GIS 门户软件平台，SuperMap iPortal 支持通过注册的方式整合<span style="font-variant-ligatures: normal; 
	 font-variant-caps: normal; orphans: 2; text-indent: 24px; widows: 2; 
	 -webkit-text-stroke-width: 0px; background-color: #ffffff; text-decoration-style: initial; 
	 text-decoration-color: initial; display: inline	 ! important ;

  <p class="myNormal"><span style="font-variant-ligatures: normal; font-variant-caps: normal; 
							 text-indent: 24px; orphans: 2; widows: 2; 
							 -webkit-text-stroke-width: 0px; text-decoration-style: initial; 
							 text-decoration-color: initial;">注册到 iPortal 中的服务可</span><span style="font-variant-ligatures: normal; font-variant-caps: normal; 
					 text-indent: 24px;">供各内置 WebApps 使用或供其他 Web GIS 应用系统调用。</span>下面详细介绍服务注册功能以及 iPortal 中服务资源的使用。</p>
  <h2><a name="registerService" id="registerService"></a>服务注册</h2>
  <p class="myNormal" style="color: #000000;">在 iPortal 中注册服务要求用户是“创建者”类型且拥有注册服务权限的角色，如 PORTAL_USER、DATA_CENTER（iPortal的用户类型及权限划分，详见：<a class="contentpagehyperlink" href="../../iportal_management/Portal_Security/RolesandPermissions.htm">角色、用户类型和权限</a>）。</p>
  <p class="myNormal" style="color: #000000;">为了对注册到门户中的多源服务进行统一的管理、对外提供统一的服务访问地址，同时也出于保护原始服务地址的考虑，iPortal 提供了<a class="contentpagehyperlink" href="../../iportal_management/Portal_config/serviceProxy_config/Service_Proxy_Config.htm">服务代理</a>功能（默认启用<a class="contentpagehyperlink" href="../../iportal_management/Portal_config/serviceProxy_config/Built_in_Proxy/serviceProxy_config.htm">内置的服务代理</a>，您也可以选择<a class="contentpagehyperlink" href="../../iportal_management/Portal_config/serviceProxy_config/Proxy_Independent/Proxy_Standalone.htm">独立部署的服务代理</a>）。iPortal 默认使用 http 协议启用服务代理，若注册的原始服务采用 https 协议，代理后的地址将以 http 协议启用。使用 https 协议启用服务代理，参见：<a class="contentpagehyperlink" href="../../iportal_management/Portal_config/serviceProxy_config/Built_in_Proxy/serviceProxy_config.htm#代理服务以_HTTPS_协议启用配置">配置服务代理以 HTTPS 协议启用</a>。</p>

  <p class="myNormal"> </p>
  <h3 style="font-weight: bold;">注册步骤</h3>
  <p class="myNormal">登录 iPortal，点击“资源中心”&gt;“服务”进入服务资源页面，点击右侧的“注册服务”进入注册页面，执行以下操作；</p>
  <ol class="ListInParagraph" style="list-style: decimal;">
    <li class="ListInParagraph">选择服务类型</li>
    <li class="ListInParagraph">填写服务地址，参见：<a class="contentpagehyperlink" href="#serviceTypes">SuperMap iPortal 支持注册的服务类型及注册地址格式</a></li>
    <li class="ListInParagraph">填写服务名称</li>
    <li class="ListInParagraph">若注册受保护的服务，需勾选“指定凭据”，并在下拉列表中选择与受保护服务匹配的凭据。若未创建凭据，点击下拉列表底部的“添加凭据”，完成<a class="contentpagehyperlink" href="../personalCenter/my_credentials.htm#add_Credentials">凭据的创建</a>。</li>
    <li class="ListInParagraph">
      <p>指定服务的存放目录（可选）：</p>
      <p>如果管理员已经在门户中开启并配置了服务资源的<a class="contentpagehyperlink" href="../../iportal_management/Portal_config/customDirectorySetting.htm">自定义目录</a>，您可以将注册的服务添加到自定义目录中。需要注意的是，同一个服务支持被添加到不同的目录结构树中，但一个目录结构树中，同一个服务只能被添加一次。</p>
    </li>
    <li class="ListInParagraph">
      <p>设置服务的元数据信息：</p>
      iPortal 会自动解析该服务的元数据信息，如：标签、描述信息、服务参照系信息（坐标系和投影类型）、服务标识信息（覆盖区域）、服务范围信息、服务提供者的基本信息等。对于没有元信息或没有解析出的元信息项您可以手动添加，也可以手动修改已解析出来的元信息，如您可以设置服务的自定义预览地址，设置后可通过该地址查看服务。完成后，点击“下一步“；
    </li>
    <li class="ListInParagraph">
      <p>设置服务的共享方式：</p>
      <p>默认的共享方式为公开可检索，即iPortal 的所有用户（包含匿名用户）都可以检索到该服务。您也可以将服务设置为公开可见，分享给 iPortal 的所有用户（包含匿名用户）可访问；或设置为私有，除本人外所有用户都无法查询或访问；或分享给指定部门（默认未开启），指定部门内的成员都可以查询和访问该服务；也可分享给指定的群组，组内的成员都可以查询和访问该服务；或分享给指定的部分用户，除本人外只有这些用户可以查询和访问该服务。设置详情，参见：<a class="contentpagehyperlink" href="../share_resource/share_serviceResource.htm">共享服务</a>。</p>
    </li>
    <li class="ListInParagraph">
      <p>点击“注册”按钮完成服务注册。</p>
    </li>
  </ol>
  <p class="myNormal"> </p>
  <p class="myNormal">注册服务时，会检测服务是否真实可用，若服务不可用，即不支持注册。唯一不同的是，注册“其他”类型的服务时，可禁用服务在线检测，也允许用户自定义设置需要检测的服务地址。</p>
  <p class="myNormal">注册完成后的服务项会显示在“资源中心”&gt;“服务”&gt;“我的服务”页面。在”我的服务“页面，您可查看到该服务的可访问状态（iPortal对注册进来的服务支持动态在线/离线校验，默认更新时间为30分钟。管理员可根据需要修改此项配置，详见：<a class="contentpagehyperlink" href="../../iportal_management/servicesmanage.htm#服务动态更新时间设置">服务动态更新时间设置</a>。），也可以查看/编辑详情、修改<a class="contentpagehyperlink" href="../share_resource/share_serviceResource.htm">共享设置</a>或<a class="contentpagehyperlink" href="../delete_resource/delete_resource.htm">删除此资源</a>，还可以<a class="contentpagehyperlink" href="../my_content/myServices.htm#更新服务">更新服务</a>的地址与元信息。</p>
  <p class="myNormal">iPortal 也为管理员提供了批量注册 SuperMap iServer 服务的功能，详见：<a class="contentpagehyperlink" href="../../iportal_management/servicesmanage.htm#批量注册" style="font-weight: normal;">批量注册</a>。</p>
  <p class="myNormal">此外，若 iPortal 管理员开启了<a class="contentpagehyperlink" href="../../iportal_management/servicesmanage.htm#serviceAudit">服务审核</a>功能（默认未开启），注册到 iPortal 中的这些服务（包括托管数据发布后自动注册的服务）会进入“待审核”状态，需要管理员审核通过才能注册成功。对未审核通过的服务设置共享不会生效。</p>
  <p class="myNormal" style="color: #ff00ff;"> </p>
  <p class="myNormal" style="font-weight: bold;">注意：</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1">同一个登录用户不能重复注册相同服务地址的服务。</li>
    <li class="SymbolList1">iPortal 不支持解析 WFS2.0.0 及以上版本、WCS1.1.1 及以上版本的元数据信息。</li>
    <li class="SymbolList1">iPortal 解析出来的元数据信息中，坐标系信息出现 -1000 等异常值时，表示该服务的坐标系是用户自定义的。</li>
  </ul>
  <h2><a name="useServiceResource" id="useServiceResource"></a>使用iPortal中的服务资源</h2>
  <p class="myNormal">iPortal 中的服务资源（含注册服务和托管服务）可供用户进行<a class="contentpagehyperlink" href="../resources.htm#预览服务">预览</a>查看，可作为各内置 Web Apps 中的数据来源，也可供用户开发的 Web GIS 系统调用。具体来说：</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1"><span style="font-weight: bold;">在数据上图 WebApp 中：</span>选择底图后，可<a class="contentpagehyperlink" href="build_map/addlayer.htm#通过搜索添加">通过搜索</a>将“我的服务”中的REST 地图服务和数据服务作为图层添加到底图上；若添加的是REST 数据服务，可基于此图层继续制作您需要的<a class="contentpagehyperlink" href="build_map/modifyLayerstyle.htm">专题图</a>。</li>
  </ul>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1"><span style="font-weight: bold;">在地图工作室 WebApp 中：</span>选择底图后，可将来自 Web 和门户中的 SuerMap REST 服务、OGC WMTS 服务、OGC WMS 服务作为图层添加到底图上；若添加的是 REST 数据服务，可基于此图层继续制作您需要的<a class="contentpagehyperlink" href="../mapstudio/layerStyle.htm">专题图</a>。</li>
    <li class="SymbolList1"><span style="font-weight: bold;">在数据洞察 WebApp 中：</span>可将<a class="contentpagehyperlink" href="../iDataInsights/AddingData.htm#iPortal_data_services">REST 地图服务或 REST 数据服务添加</a>进来，基于这些服务结合您的需求，可使用<a class="contentpagehyperlink" href="../iDataInsights/CreatingView.htm">地图或图表</a>进行可视化表达，也可在地图视图中，使用<a class="contentpagehyperlink" href="../iDataInsights/Analysis.htm">分析功能</a>更深层次地发掘数据蕴藏的规律与价值。</li>
    <li class="SymbolList1"><span style="font-weight: bold;">在三维地球 WebApp 中：</span>您可将<a class="contentpagehyperlink" href="../scene_usage/EditScene.htm#addLayer">REST 三维服务以图层的形式添加</a>到场景中，基于此场景<a class="contentpagehyperlink" href="../scene_usage/drawFeature.htm">添加点线面要素</a>，或在场景中<a class="contentpagehyperlink" href="../scene_usage/controls.htm">量算、查询</a>功能，或<a class="contentpagehyperlink" href="../scene_usage/AnalyseScene.htm">三维分析</a>等。</li>
    <li class="SymbolList1"><span style="font-weight: bold;">在地图大屏 WebApp 中</span>，支持将公开的服务、我的服务、iServer REST 服务等作为可视化组件的数据源，详见：<a class="contentpagehyperlink" href="../mapdashboard/add_component/AddComponent.htm">添加可视化组件</a>。</li>
    <li class="SymbolList1"><span style="font-weight: bold;">在应用构建器 WebApp（Beta）中</span>，支持将公开的服务、我的服务、iServer REST 服务等作为可视化组件的数据源，详见：<a class="contentpagehyperlink" href="../builderX/addData.htm">添加数据</a>。</li>
    <li class="SymbolList1"><span style="font-weight: bold;">在您开发的Web GIS系统中调用iPortal中的服务资源</span>，详见：使用 iCient JavaScript 访问 iPortal 中的资源。</li>
  </ul>
