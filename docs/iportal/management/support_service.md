  <h2><a id="Web_打印配置"></a>Web 打印配置</h2>
  <p class="myNormal">SuperMap iPortal 支持对接 SuperMap iServer 中的 <a class="contentpagehyperlink" href="../../iS/use_iserver/usewebprinting/use_webprinting.htm">Web 打印服务</a>，实现在数据上图 WebAPP 中进行<a class="contentpagehyperlink" href="../iportal_usage/add_resource/build_map/savaandshare.htm#地图打印">地图打印</a>，将制作的地图输出为可打印的文档。iPortal 管理员可以通过以下步骤完成 Web 打印配置，以接入 iServer 中的 Web 打印功能：</p>
  <ol class="ListInParagraph" style="list-style: decimal;">
    <li class="ListInParagraph">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ Web 打印服务”选项卡；</li>
    <li class="ListInParagraph">“Web 打印服务地址”：填写对接的 iServer Web 打印服务地址，格式为：http://&lt;server&gt;:&lt;port&gt;/iserver/services/webprinting/rest/webprinting/v1；</li>
    <li class="ListInParagraph">“iPortal 根地址”（选填）：填写 <span>iServer Web 打印服务可以访问此 iPortal 的实际根地址。若 iPortal 存在多地址或内外网访问方式，请填写此参数，指定一个访问地址。<span>示例：http://&lt;server&gt;:&lt;port&gt;/{contextPath}</span></span>；</li>
    <li class="ListInParagraph">点击“保存”按钮，即可完成 Web 打印配置。</li>
  </ol>
  <p class="myNormal">配置完成后，您就可以在数据上图 WebApp 中使用 Web 打印功能了。</p>
  <h2 class="myNormal"><a id="地址匹配服务设置"></a>地址匹配服务配置</h2>
  <p class="myNormal">在数据上图 WebApp 或地图工作室 WebApp 中使用文件上传的方式添加图层时，如果源文件中通过地址表达位置信息，可以通过地址匹配服务或地理编码服务解析城市范围和地址描述，从而返回对应的地理坐标和结构化的地址详细描述。</p>
  <p class="myNormal">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 地址匹配服务”选项卡，添加地址匹配服务地址或地理编码服务 Key：</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1">SuperMap iServer 地址匹配服务：填写 SuperMap iServer 中地址匹配服务的地址，例如：http://&lt;server&gt;:&lt;port&gt;/iserver/services/addressmatch-Address/restjsr/v1/address</li>
    <li class="SymbolList1">SuperMap Online 地理编码服务：填写地理编码服务的 Key。需要注意的是，您必须携带 Key 才能调用地理编码服务 REST API ，申请地址：<a class="contentpagehyperlink" href="https://www.supermapol.com/web/mycontent/keys/register">https://www.supermapol.com/web/mycontent/keys/register</a></li>
  </ul>
  <p class="myNormal">填写完成后，点击“保存”按钮即可。</p>
  <h2><a id="POI_搜索服务设置"></a>搜索服务配置</h2>
  <p class="myNormal">数据上图 WebApp 和地图工作室 WebApp 的 POI 搜索控件，提供了查询 POI 信息的能力。</p>
  <p class="myNormal">在使用 POI 搜索功能前，管理员需配置搜索服务提供者。依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 搜索服务”选项卡，选择服务类型并填写相应参数：</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1">SuperMap iServer REST 数据服务：填写 SuperMap iServer REST 数据服务的地址和查询字段（逗号分隔），REST 数据服务地址示例：http://&lt;server&gt;:&lt;port&gt;/iserver/services/data-changchun/rest/data/datasources/Changchun/datasets/Hospital</li>
    <li class="SymbolList1">SuperMap Onine 本地搜索服务：填写 SuperMap Online 本地搜索服务的 Key。需要注意的是，您必须携带 Key 才能调用 本地搜索服务 REST API，申请地址：<a class="contentpagehyperlink" href="https://www.supermapol.com/web/mycontent/keys/register">https://www.supermapol.com/web/mycontent/keys/register</a></li>
    <li class="SymbolList1">Elasticsearch 搜索服务：填写 Elasticsearch 搜索服务地址，还可以根据需要输入查询字段和坐标字段（逗号分隔），服务地址示例：http://{ip}:{port}/index/type</li>
  </ul>
  <h2><a id="空间分析服务配置"></a>空间分析服务配置</h2>
  <p class="myNormal">数据洞察（DataInsights）目前提供标准空间分析和分布式空间分析两种空间分析工具。要使用基于 iServer 的标准空间分析工具和分布式空间分析工具，需要先配置好分析服务地址。</p>
  <h3>标准空间分析服务</h3>
  <p class="myNormal">标准空间分析支持<span style="font-weight: bold;">客户端空间分析</span>和<span style="font-weight: bold;">基于 SuperMap iServer 的空间分析</span>两种方式，其中，客户端空间分析无需进行配置，即可使用；基于 iServer 的空间分析功能需要使用 iServer 的空间分析工具，在 iPortal 中添加 iServer 分析服务器的步骤如下：</p>
  <p class="myNormal">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 空间分析服务”选项卡，在“标准空间分析服务”下填写参数：</p>
  <ol class="ListInParagraph" style="list-style:decimal;">
    <li class="ListInParagraph">服务地址：输入 iServer 空间分析服务地址，例如：http://ip:port/iserver/services/spatialanalyst-xx/restjsr/spatialanalyst</li>
    <li class="ListInParagraph">令牌：如果 iServer 中发布的空间分析服务是受保护的，则需输入 iServer 的令牌，令牌的获取方法请查看：<a class="contentpagehyperlink" href="../../Subject_introduce/Security/config_role/token/AcquiringSuperMap_Token.htm">获取 Token</a></li>
    <li class="ListInParagraph">单击“保存”按钮即可生效</li>
  </ol>
  <h3>分布式空间分析服务</h3>
  <p class="myNormal">数据洞察的分布式空间分析功能是基于 SuperMap iServer 的分布式分析服务实现的，因此，需要添加一个启用了分布式分析服务的 iServer 服务器用于数据洞察进行分布式空间分析。</p>
  <p class="myNormal">在 SuperMap iServer 中开启分布式分析服务后，iPortal 中添加 iServer 分布式分析服务器的步骤如下：</p>
  <p class="myNormal">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 空间分析服务”选项卡，在“分布式空间分析服务”下填写参数：</p>
  <ol class="ListInParagraph" style="list-style:decimal;">
    <li class="ListInParagraph"><span class="myNormal">服务地址：输入 iServer 分布式分析服务地址，例如<span style="color: rgb(63, 63, 63);">：</span>http://ip:port/iserver/services/distributedanalyst/rest/v1/jobs</span></li>
    <li class="ListInParagraph">令牌：如果 iServer 分布式分析服务是受保<span class="myNormal">护的，则需要输入 iServer 的令牌，令牌的具体获取方法请查看：</span><a class="contentpagehyperlink" href="../../Subject_introduce/Security/config_role/token/AcquiringSuperMap_Token.htm">获取 Token</a></li>
    <li class="ListInParagraph">单击“保存”按钮即可生效</li>
  </ol>
  <p class="myNormal"><span style="font-weight: bold;">注意</span>：</p>
  <ol class="ListInParagraph" style="list-style: decimal;">
    <li class="ListInParagraph">对 Excel、CSV、GeoJSON 格式数据进行分布式空间分析时，iPortal 托管的 iServer 服务器、配置用于关系型存储的 iServer 服务器 、用于分布式分析的 iServer 服务器，三者需要为同一个 SuperMap iServer</li>
    <li class="ListInParagraph">对注册的 HDFS 数据进行分布式分析时，iPortal 托管的 iServer 服务器和用于分布式分析的 iServer服务器，两者需要为同一个 SuperMap iServer</li>
  </ol>
  <p class="myNormal">有关托管服务器的配置，请参见：<a class="contentpagehyperlink" href="Services_hosting/hostedServices_config.htm">托管服务器</a>；有关关系型存储的配置，请参见：<a class="contentpagehyperlink" href="Services_hosting/Storage_Config.htm#关系型存储">托管数据存储配置</a>。</p>
  <h2><a id="模型数据集服务配置"></a>模型数据集服务配置</h2>
  <p class="myNormal">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 模型数据集服务”选项卡：</p>
  <ol>
    <li class="ListInParagraph">“模型数据集地址”：填写模型数据集的 iServer 服务地址，格式为：http://ip:port/iserver/services/{serviceId}/rest/data/datasources/{datasourceName}/datasets/{datasetName} ；</li>
    <li class="ListInParagraph">“iServer 管理员账户名”：填写模型数据集的 iServer 服务器管理员账号；</li>
    <li class="ListInParagraph">“iServer 管理员账户密码”：填写模型数据集的 iServer 服务器管理员密码；</li>
    <li class="ListInParagraph">“是否云原生”：模型数据集的 iServer 服务器是否为云原生环境；</li>
    <li class="ListInParagraph">点击“保存”按钮，即可完成模型数据集服务配置。</li>
  </ol>
  <p class="myNormal">配置完成后，您就可以在三维地理设计 WebApp 中使用模型数据集服务了。</p>
  <h2><a name="影像处理服务配置"></a><a id="实时路况服务配置"></a>实时路况服务配置</h2>
  <p class="myNormal">以管理员身份登录门户首页，依次点击“管理”&gt;“应用中心配置”&gt;“支撑服务”，选择“ 实时路况服务”选项卡：</p>
  <ol>
    <li class="ListInParagraph">“申请地址”：<span>登录SuperMap Online，申请实时路况服务</span>；</li>
    <li class="ListInParagraph">“服务密钥”：<span>填写服务 Key</span>；</li>
    <li class="ListInParagraph">点击“保存”按钮，即可完成实时路况服务配置。</li>
  </ol>
  <p class="myNormal">配置完成后，您就可以在数据上图 WebApp 中使用实时路况服务了。</p>
