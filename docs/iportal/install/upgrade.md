  <h2>从 iPortal 8C 及以前版本升级</h2>
  <p class="myNormal">如果您正在使用的是 SuperMap iPortal 7C 系列版本或 8C 版本，需要升级至 SuperMap iPortal 8C SP1 版本或更新版本，请联系 SuperMap iPortal 团队，获取产品升级服务。联系电话：010-59896716</p>
  <p class="myNormal">从 SuperMap iPortal 8C SP1 版本开始，iPortal 实现了独立部署，不再内置 SuperMap iServer 应用服务器，针对先前版本的门户升级，主要包括以下几方面：</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1">门户内容的升级：包括在线制作的地图、注册的服务、创建的群组、托管的数据、托管的服务、门户管理员发布的服务、监控数据等</li>
    <li class="SymbolList1">用户账户数据的升级</li>
    <li class="SymbolList1">门户配置信息升级</li>
    <li class="SymbolList1">前端门户定制内容升级</li>
  </ul>
  <h2>从 iPortal 8C SP1 及更新版本升级</h2>
  <p class="myNormal">SuperMap iPorta 自 8C SP1 版本起，在产品包中提供了用于产品升级的脚本文件，可以通过该脚本文件快速完成 iPortal 版本的升级。升级脚本文件位于【SuperMap iPortal 安装目录】\support\upgrade 目录下，其中的 <span style="font-weight: bold;">autoUpgrade.bat 文件</span>（用于 Windows 平台）和<span style="font-weight: bold;"> autoUpgrade.sh 文件</span>（用于 Linux 平台）可以帮助您自动进行 iPortal 中门户内容、配置文件、图片等绝大部分内容的升级。下面介绍升级脚本的具体使用方法：</p>
  <ol class="ListInParagraph" style="list-style: decimal;">
    <li class="ListInParagraph">在升级 iPortal 前，请先对旧版 iPortal 进行备份，以避免可能发生的意外。</li>
    <li class="ListInParagraph">如果您正在运行待升级的旧版 iPortal，请先停止该服务。</li>
    <li class="ListInParagraph">将用于替代旧版 iPortal 的新版 iPortal 产品包解压缩。</li>
    <li class="ListInParagraph">编辑 %SuperMap iPortal_HOME%\support\upgrade\autoUpgrade.bat 文件（Linux 系统下对应为 autoUpgrade.sh 文件）中 OLD_PKG 和 NEW_PKG 变量并保存，示例如下：</li>
  </ol>
  <p class="myCodeP">set &quot;OLD_PKG=D:\supermap_iportal_8.0.1a_win64_zip&quot;</p>
  <p class="myCodeP">set &quot;NEW_PKG=D:\supermap-iportal-9.1.0-win64-zip&quot;</p>
  <ul class="SymbolList1" style="list-style: disc;">
    <li class="SymbolList1">OLD_PKG：指向旧版 iPortal 根目录的绝对路径，不能以 &quot;/&quot; 或 &quot;\&quot; 结尾</li>
    <li class="SymbolList1">NEW_PKG：指向新版 iPortal 根目录的绝对路径，不能以 &quot;/&quot; 或 &quot;\&quot; 结尾</li>
  </ul>
  <ol class="ListInParagraph" start="5" style="list-style: decimal;">
    <li class="ListInParagraph">运行升级脚本。Windows 系统中双击运行 autoUpgrade.bat，Linux 系统中切换到 %SuperMap iPortal_HOME%\support\upgrade 目录后，输入如下命令：</li>
  </ol>
  <p class="myCodeP">./autoUpgrade.sh</p>
  <ol class="ListInParagraph" start="6" style="list-style: decimal;">
    <li class="ListInParagraph">脚本运行完毕后，旧版 iPortal 中的门户数据、配置文件、首页定制等内容已迁移至新版 iPortal 中，完成了产品的升级。运行 {IPORTAL_HOME_NEW}\bin\startup.bat（Linux 系统中为 startup.sh），即可使用新版 iPortal。</li>
  </ol>
  <p class="myNormal">注：欲了解该脚本在升级过程中做的具体操作，可参阅：%SuperMap iPortal_HOME%\support\upgrade\readme_zh_cn.txt。</p>
  <h3>需要手动升级的内容</h3>
  <p class="myNormal">使用升级脚本后，以下这些配置还需要手动进行升级：</p>
  <p class="myCodeP">&lt;bean id=&quot;esSetting&quot; class=&quot;com.supermap.iportal.search.commontypes.ESSetting&quot;&gt;</p>
  <p class="myCodeP">    ...</p>
  <p class="myCodeP">    &lt;property name=&quot;port&quot; value=&quot;<span style="font-weight: bold;">39200</span>&quot;&gt;&lt;/property&gt;  </p>
  <p class="myCodeP">    ...</p>
  <p class="myCodeP">&lt;/bean&gt;</p>
  <ol class="ListInParagraph" style="list-style: decimal;">
    <li class="ListInParagraph">根据“旧版 iPortal” 的 {IPORTAL_HOME_OLD}/conf/server.xml，去编辑配置“新版 iPortal”的 {IPORTAL_HOME_NEW}/conf/server.xml，尤其是 Server port 、 connector port 以及其它自定义的配置。</li>
    <li class="ListInParagraph">“新版 iPortal”的 iportal-storage.xml 文件需要通过界面来配置：管理-&gt;服务管理-&gt;托管服务器-&gt;托管数据配置，该界面的参数需要依据”旧版 iPortal“的 {IPORTAL_HOME_OLD}/webapps/iportal/WEB-INF/iportal-storage.xml 的内容进行配置，关于如何配置请参见：<a class="contentpagehyperlink" href="../iportal_management/Services_hosting/hostedServices_config.htm">托管服务配置</a>。</li>
    <li class="ListInParagraph">若”旧版 iPortal“进行过<a class="contentpagehyperlink" href="../iportal_management/customPortal/themeManage.htm">主题的定制</a>，那么需要：<br/>
      1） 将“旧版 iPortal”中的新增的自定义主题 {IPORTAL_HOME_OLD}/webapps/iportal/resources/web-ui/config/themes/ThemeSetting_**.json 文件拷贝至新版 iPortal {IPORTAL_HOME_NEW}/webapps/iportal/resources/web-ui/config/themes 目录中。<br/>
      2） 比对自定义主题文件 ThemeSetting_**.json 和“新版 iPortal”的 {IPORTAL_HOME_NEW}/webapps/iportal/resources/web-ui/config/themes/ThemeSetting_blue.json 文件结构，将 ThemeSetting_blue.json 文件中新增的 json 对象拷贝到 ThemeSetting_**.json 中，并赋予自定义色值进行覆盖。<br/>
      3）参考“旧版 iPortal”的 {IPORTAL_HOME_OLD}/webapps/iportal/resources/web-ui/config/themes/themes.json 去编辑配置”新版iPortal“的{IPORTAL_HOME_NEW}/webapps/iportal/resources/web-ui/config/themes/themes.json，或通过 管理-&gt;站点配置-&gt;站点定制-&gt;<a href="../iportal_management/customPortal/themeManage.htm">主题管理</a> 以界面方式重新导入自定义的主题。</li>
    <li class="ListInParagraph">若“旧版 iPortal”进行过<a class="contentpagehyperlink" href="../iportal_management/Portal_config/multiLanguage.htm">多语言定制</a>，增加了新的语言种类，那么需要：<br/>
      1）对比 {IPORTAL_HOME_NEW}/webapps/iportal/resources/web-ui/config/Language_**.json（新增加的语言配置文件） 和  {IPORTAL_HOME_NEW}/webapps/iportal/resources/web-ui/config/Language_zh.json 文件。将 Language_zh.json 文件中新增的 json 对象拷贝到 Language_**.json 文件中，并进行翻译和覆盖。<br/>
      2）对比 {IPORTAL_HOME_NEW}/webapps/iportal/resources/admin-ui/lang/Language_**.json（新增加的语言配置文件） 和  {IPORTAL_HOME_NEW}/webapps/iportal/resources/admin-ui/lang/Language_zh.json 文件。将 Language_zh.json 文件中新增的 json 对象拷贝到 Language_**.json 文件中，并进行翻译和覆盖。</li>
    <li class="ListInParagraph">若“旧版 iPortal”配置过 <a class="contentpagehyperlink" href="../iPortal_DevelopGuide/Custom_iPortal/integra_userAccount/CAS_integra_userAccount.htm">CAS 单点登录</a>，那么需要参考旧版的 {IPORTAL_HOME_OLD}/webapps/iportal/WEB-INF/shiro.ini，去编辑配置“新版 iPortal”的 {IPORTAL_HOME_NEW}/webapps/iportal/WEB-INF/shiro.ini。</li>
    <li class="ListInParagraph">若要升级 ElasticSearch 中存储的门户资源，需要编辑“新版 iPortal”的 {IPORTAL_HOME_NEW}/webapps/iportal/WEB-INF/iportal.xml， 修改 IportalConfig-&gt;modulesConfig-&gt;searchModuleConfig-&gt;reindexAllResources 值为true。</li>
    <li class="ListInParagraph"><span>若“旧版 iPortal”配置的外置ElasticSearch （7.17以下版本），需要参考 ElasticSearch 官方文档将外置 ElasticSearch 升级至 8.19.x 版本（兼容7.17.x版本）</span></li>
    <li class="ListInParagraph">若希望对版权信息内版本号进行变更，可通过 管理 -&gt; 站点配置 -&gt; 站点定制 -&gt; 常规内容-&gt; <a href="../iportal_management/customPortal/customHome.htm#generalSettings">版权信息</a> 进行配置。</li>
    <li class="ListInParagraph">若“旧版 iPortal” 为 SuperMap iPortal 10i 以下版本，您在“旧版iPortal”中定制的首页将无法被升级，推荐您使用“新版 iPortal” <a class="contentpagehyperlink" href="../iportal_management/customPortal/customHome.htm">首页定制</a>功能重新定制您的 iPortal 首页。</li>
    <li class="ListInParagraph">若“旧版 iPortal” 进行过后台管理界面的<a class="contentpagehyperlink" href="../iPortal_DevelopGuide/Custom_iPortal/Admin_UI_custom/Admin_UI_custom.htm">扩展开发</a>，那么需要参考旧版的 {IPORTAL_HOME_OLD}/webapps/iportal/WEB-INF/urlrewrite.xml，去编辑配置“新版iPortal”的{IPORTAL_HOME_NEW}/webapps/iportal/WEB-INF/urlrewrite.xml，将旧版中的自定义配置项拷贝至新版中。</li>
    <li class="ListInParagraph">若“旧版 iPortal”为 SuperMap iPortal 11i 以下的版本，在升级为 11i 及以上的版本时，需编辑“新版 iPortal”的 {IPORTAL_HOME_NEW}/webapps/iportal/WEB-INF/config/search/WEB-INF/search-beans.xml，将 ElasticSearch 搜索服务器的端口由 TCP 端口39300 修改为 HTTP 端口39200：
      <ul class="SymbolList2" style="list-style: circle;">
        <li class="SymbolList2">port：Elasticsearch 的 HTTP 协议通讯端口号。默认值：39200。</li>
      </ul>
    </li>
    <li class="ListInParagraph">若“旧版 iPortal”编辑或升级过 iportal-ehcache.xml 文件（位于目录 {IPORTAL_HOME_OLD}/webapps/iportal/WEB-INF ），那么需要参考旧版配置编辑“新版 iPortal”的 {IPORTAL_HOME_NEW}/webapps/iportal/WEB-INF/ iportal-ehcache.xml 文件。</li>
    <li class="ListInParagraph"><span>若“旧版 iPortal” 编辑过内置代理的 web.xml 文件(位于目录 {IPORTAL_HOME_OLD}/webapps/iportal/WEB-INF/config/proxy/WEB-INF/web.xml )，需要重新去编辑配置新版 iPortal 的 {IPORTAL_HOME_NEW}/iportal/webapps/iportal/WEB-INF/config/proxy/WEB-INF/web.xml</span></li>
  </ol>
