  <p class="myNormal" style="font-weight: normal;">SuperMap iServer 服务器会对客户端 HTTP 请求做出响应，客户端执行 GET 请求，获得资源的表述，客户端执行 PUT、POST、DELETE 请求，获得操作结果的表述。表述可以有多种格式，如 html、xml、json、rjson、jsonp、png、bmp、gif 、jpg、jpeg、classic、leaflet、openlayers3、mapboxgl 等，其中 png、bmp、gif、jpg 及 jpeg 是图片资源特有的表述格式，classic、leaflet、openlayers3、mapboxgl 是 map 资源和 layer 资源支持的表述格式。</p>
  <p class="myNormal">表述的格式在 HTTP 请求的 URI 中指定，形式是在 URI 后面加上.&lt;format&gt;。例如对 http://supermapiserver:8090/iserver/services/map-changchun/rest/maps.json 执行 GET 请求，就会返回 json 格式的 maps 资源的表述。</p>
  <p class="myNormal">如果不指定&lt;format&gt;，SuperMap iServer 服务器以默认的表述格式返回相应表述，默认返回 html 格式的表述。</p>
  <p class="myNormal">注意：获取图片资源的表述出错时，如果是 Debug 模式（在 AppContext.xml 中 REST 配置项的 debug 字段来设置，debug 为 true 表示为 Debug 模式，默认为 true），出错信息以 html 格式返回，否则以 json 格式返回。</p>

  <h2><a name="html" id="html"></a>html</h2>
  <p class="myNormal">默认的非图片资源表述格式。返回一个 HTML 页面。HTML 可以容易地表达一个列表，而且可以用标准属性（class）表达列表的种类。</p>
  <p class="myNormal">例如，要获得 maps 服务的 HTML 表述，可以用以下 URI：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/services/map-changchun/rest/maps.html</p>
  <p class="myNormal">因为 html 是默认的非图片资源表述格式，所以也可以不指定，直接使用如下 URI：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/services/map-changchun/rest/maps</p>

  <h2><a name="kml" id="kml"></a>kml</h2>
  <p class="myNormal">返回一个 KML 格式的页面，当资源是 map、 features、 feature 时有效。 KML（Keyhole Markup Language）是 GoolgleEarth 的文件格式，是一个基于 XML 语法和文件格式的文件，用来描述和保存如点、线、图片、折线等地理信息要素并在 Google Earth、 Google Map 等应用系统中显示。</p>
  <p class="myNormal">例如，要获得 features 服务的 KML 表述，可以用以下 URI：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/services/data-changchun/rest/data/datasources/Changchun/datasets/Frame/features.kml</p>
  <p class="myNormal">返回的示例如下所示：</p>
  <p class="myCodeP">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot; standalone=&quot;no&quot;?&gt;</p>
  <p class="myCodeP">  &lt;kml&gt;</p>
  <p class="myCodeP">    &lt;Document&gt;</p>
  <p class="myCodeP">      &lt;NetworkLink&gt;</p>
  <p class="myCodeP">        &lt;name&gt;0-1-0&lt;/name&gt;</p>
  <p class="myCodeP">        &lt;Link&gt;</p>
  <p class="myCodeP">          &lt;href&gt;http://supermapiserver:8090/iserver/services/data-changchun/rest/data/feature/0-1-0.kml&lt;/href&gt;</p>
  <p class="myCodeP">        &lt;/Link&gt;</p>
  <p class="myCodeP">      &lt;/NetworkLink&gt;</p>
  <p class="myCodeP">      &lt;NetworkLink&gt;</p>
  <p class="myCodeP">        &lt;name&gt;0-1-1&lt;/name&gt;</p>

  <p class="myCodeP">        &lt;/Link&gt;</p>
  <p class="myCodeP">      &lt;/NetworkLink&gt;</p>

  <p class="uriStyle">http://supermapiserver:8090/iserver/services/map-changchun/rest/maps/长春市区图.kml</p>
  <h2><a name="georss" id="georss"></a>georss</h2>
  <p class="myNormal">返回一个 GeoRSS 格式的页面，当资源是 features、feature 时有效。GeoRSS 是 OGC 在 GML（地理标记语言） 基础上推出的描述地理信息的规范，它使用 XML 语言存储和传输地理数据，用来描述和保存如点、线、面等地理信息要素。GeoRSS 提供了 GeoRSS Simple 和 GeoRSS GML Profile 两种形式，SuperMap iServer 目前支持 GeoRSS Simple 表述。GeoRSS 格式的地理要素信息可以通过 Virtual Earth 等应用系统显示。</p>
  <p class="myNormal">例如，data-changchun/rest 服务中有一个名为 Park 的数据集，要获取其索引号为 0-0-0 的要素的 georss 表述，可以用以下 URI（有关 feature 资源的请求 URI，请参见 <a href="root/data/datasources/datasource/datasets/dataset/features/feature/feature.htm">feature 资源</a>介绍）：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/services/data-changchun/rest/data/feature/0-0-0.georss</p>
  <p class="myNormal">返回的示例如下所示：</p>

  <p class="myNormal">返回请求的 JPEG 格式图片，仅当资源是图片时有效。</p>
  <p class="myNormal">例如，要获得名为长春市区图的地图的当前显示，格式为 JPEG（*.jpg），可以用以下 URI：</p>

  <h2 class="uriStyle"><a name="WebP"></a>WebP</h2>
  <p class="myNormal">返回请求的 WebP 格式图片，仅当资源是图片时有效。WebP 是谷歌提出的一种既提供有损压缩也提供无损压缩的图片格式，WebP 无损压缩图片的大小小于相同图像质量的 PNG 格式的图像。目前地图服务的 image，tileImage，entireimage，overview，symbol，zxyTileImage 资源均支持通过 WebP 表述浏览。</p>
  <p class="myNormal">例如，要获得名为长春市区图的地图的当前显示，格式为 WebP（*.webp），可以用以下 URI：</p>

  <p class="uriStyle" style="font-weight: bold;">注：在切图时，栅格、影像数据建议使用 WebP 格式，矢量数据建议使用 PNG 格式。</p>

  <p class="uriStyle">http://supermapiserver:8090/iserver/representations/leaflet/map.html?serviceName=map-World&amp;mapName=World</p>
  <h2><a name="mapboxgl"></a>mapboxgl</h2>
  <p class="myNormal">当资源是 map 时，且地图的投影坐标系为3857、4326、4490、4214、4610 时，mapboxgl 表述有效。mapboxgl 表述是针对 JavaScript 客户端的应用，实现了简单的地图功能，如缩放、量算等。mapboxgl 表述仅支持浏览地图矢量要素。</p>
  <p class="myNormal">例如，要获得名为 China 的地图的 mapboxgl 表述，可以用以下 URI：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/representations/mapboxgl/map.html?serviceName=map-World&amp;mapName=World</p>
  <h2 class="uriStyle"><a name="WebGL3D"></a>WebGL3D</h2>
  <p class="myNormal">当资源是 map，且地图不是平面坐标系时，WebGL3D 表述有效。WebGL3D实现了简单的三维地图功能，如缩放、旋转等。</p>
  <p class="myNormal">例如，要获得名为 World 的地图的 WebGL3D 表述，可以用以下 URI：</p>
  <p class="uriStyle">http://supermapiserver:8090/iserver/representations/webgl3d/map.html?serviceName=map-World&amp;mapName=World</p>
  <h2><a name="FlatGeobuf"></a>FlatGeobuf</h2>
  <p class="myNormal">当资源是 featureResults、featureResult、queryResults、queryResult、connectedEdges、traceDown、traceUp、geometryOverlayResult (geometry)、geometryBufferResult (geometry)、datasetBufferResult、datasetOverlayResult、datasetIsolineResult、datasetIsoregionResult、datasetThiessenPolygonResult、geometryBufferResult (spatialAnalyst)、geometryOverlayResult (spatialAnalyst)、geometryIsolineResult、 geometryIsoregionResult、 geometryThiessenPolygonResult 时，FlatGeobuf 表述有效。FlatGeobuf 是一种基于 flatbuffers，能容纳简单要素（Simple Features）规范下数据的二进制编码格式，具有文件较小，能够分段传输和局部加载的特点。FlatGeobuf 表述可以通过 URI.fgb 形式获得。</p>
  <p class="myNormal">例如，要获得 World 数据服务中 Capitals 数据集 ID 为1-3 的要素查询结果的 FlatGeobuf 表述，可以用以下 URI：</p>
  <p class="myNormal">         http://supermapiserver:8090/iserver/services/data-World/rest/data/featureResults/d935257d1b284a3c9c187f65d8417b12_71377342b8b742deb45ea5fd4c58bf45.fgb</p>
  <p class="myNormal">可获得名为 d935257d1b284a3c9c187f65d8417b12_71377342b8b742deb45ea5fd4c58bf45.fgb 的文件，可用 QGIS 等软件打开进行使用。</p>
