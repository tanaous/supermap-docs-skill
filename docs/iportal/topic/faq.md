      <li class="FAQList">启动 SuperMap iPortal 服务时，环境检查不通过的几种可能原因及解决方法。<a class="contentpagehyperlink" href="#toc"><img alt="" border="0" height="16" src="../../assets/images/FAQ/gray_up.gif" style="border: none;" width="15"/></a></li>

    <p>答：SuperMap iPortal 服务需要配置 JRE 1.8 或以上版本，需要配置 SuperMap iPortal 的使用许可。</p>
    <p>在 %SuperMap iPortal_HOME%\bin 目录下使用“iportal.bat -v”（Linux/Unix 平台使用 “./iportal.sh -v”）可以查看当前 SuperMap iPortal 的版本及配置信息。可能原因及解决方法如下：</p>
    <p>（1） SuperMap iPortal 许可配置不正确，请确保使用许可已正确配置。许可的配置请参见<a class="contentpagehyperlink" href="../Install_Uninstal_iportal/License_config/License_Config.htm">许可配置说明</a>。</p>
    <p>（2） JRE/JDK 版本过低或者不兼容。SuperMap iPortal 支持1.8以上版本的 JRE/JDK。</p>

      <li class="FAQList">为什么运行 SuperMap iPortal 过程中报如下错误：“java.lang.OutOfMemoryError: unable to create new native thread”？<a class="contentpagehyperlink" href="#toc"><img alt="" border="0" height="16" src="../../assets/images/FAQ/gray_up.gif" style="border: none;" width="15"/></a></li>

    <p class="myCodeP">insert into user_infos(name,nickname) values(&quot;admin&quot;,&quot;admin&quot;);</p>
    <p>（针对 SuperMap iPortal 8C SP1之前的版本，您还需确保 MySQL 数据库中的 servicerooturls 表是空的，如果有记录，需要删除 ），然后重启 iPortal 服务即可。</p>
    <p><span style="font-weight: bold;">需要注意的是</span>：如果您已经使用了 iPortal 创建地图、注册服务等，切换至 MySQL 数据库时如果想保留之前的门户数据信息，那么还需要同步 iportal.db 中的所有门户数据信息。</p>
    <p> </p>
    <ol class="Decimal" start="41">
      <li class="FAQList"><a id="MySQL8.0配置"></a>如果用户使用 MySQL 8.0 及以上版本的数据库时，启动 iPortal 失败并报如下类似错误：“Current charset is GBK. If password has been set using other charset, consider using option &#39;passwordCharacterEncodng&#39;”，如何解决？<a class="contentpagehyperlink" href="#toc"><img alt="" border="0" height="16" src="../../assets/images/FAQ/gray_up.gif" style="border: none;" width="15"/></a></li>
    </ol>
    <p>答：这是由于 MySQL 8.0 及以上版本的默认密码加密规则从 mysql_native_password 变更为 caching_sha2_password 而导致 iPortal 启动失败。您可以在连接到 MySQL 后，执行以下命令以更改密码加密规则：</p>
    <p class="myCodeP">use mysql;</p>
    <p class="myCodeP">alter user &#39;<span style="font-weight: bold;">username</span>&#39;@&#39;<span style="font-weight: bold;">%</span>&#39; identified with mysql_native_password by &#39;<span style="font-weight: bold;">password</span>&#39;;</p>
    <p>其中 username 为用户名，%为 user 表中设置的 host ，password 为用户密码，需根据 MySQL 数据库实际配置填写。配置完成后重启 iPortal 即可。</p>
    <p> </p>
    <ol class="Decimal" start="42">
      <li class="FAQList"><span><a id="密钥设置服务过多"></a></span><span>如果用户在设置</span>密钥时选择的服务数目较多<span style="text-decoration: none; ">，添加密钥无响应，且报如下类似错误：“Error updating database.  Cause: java.sql.SQLDataException: Data too long for column &#39;serviceIds&#39; at row 1”，如何解决？</span><a class="contentpagehyperlink" href="#toc"><img alt="" border="0" height="16" src="../../assets/images/FAQ/gray_up.gif" style="border: none;" width="15"/></a></li>
    </ol>
    <p>答：这是由于密钥对应的服务数目过多时，服务 ID 的长度超出了数据库中对应字段的字符长度限制。根据您配置的门户数据库不同（MySQL/Oracle/PostgreSQL），可尝试通过以下方法解决：</p>
    <p>（1）MySQL 数据库：在连接到 iPortal 门户数据库后，执行以下 SQL 语句，将 serviceIds 的字段类型修改为 text：</p>
    <p class="myCodeP">alter table keycodes modify serviceIds text not null;</p>
    <p>（2）Oracle 数据库：在连接到 iPortal 门户数据库后，执行以下 SQL 语句，将 serviceIds 的字段类型修改为 clob：</p>
    <p class="myCodeP">alter table &quot;keycodes&quot; rename column &quot;serviceIds&quot; to &quot;serviceIds_temp&quot;;</p>
    <p class="myCodeP">alter table &quot;keycodes&quot; add &quot;serviceIds&quot; clob;</p>
    <p class="myCodeP">update &quot;keycodes&quot; set &quot;serviceIds&quot;=trim(&quot;serviceIds_temp&quot;);</p>
    <p class="myCodeP">alter table &quot;keycodes&quot; drop column &quot;serviceIds_temp&quot;;</p>
    <p>（3）PostgreSQL数据库：在连接到 iPortal 门户数据库 后，执行以下 SQL 语句，将 serviceIds 的字段类型修改为 text：</p>
    <p class="myCodeP">ALTER TABLE keycodes ALTER COLUMN serviceIds TYPE text;</p>
    <p> </p>
    <ol class="Decimal" start="43">
      <li class="FAQList"><a id="监控不到服务器信息"></a>SuperMap iPortal 8C SP1 及之前的版本，使用了服务器监控功能，升级至 8C SP2 及之后的版本时，监控不到服务器信息了，该如何解决？<a class="contentpagehyperlink" href="#toc"><img alt="" border="0" height="16" src="../../assets/images/FAQ/gray_up.gif" style="border: none;" width="15"/></a></li>
    </ol>
    <p>答：可尝试在“监控服务器”-&gt;“监控配置”页面重新输入密码后，再保存</p>
